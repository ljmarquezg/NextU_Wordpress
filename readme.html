  • 1. Crear espacio de trabajo arrow_drop_down

      1. Instalar Xampp en el computador:
        Utilizar la ruta predeterminada: C:\xampp

        https://www.apachefriends.org/es/index.html

      2. Abrir la cónsola de xampp con permisos de administrador

      3. Instalar los Apache y MySql como servicios de windows:
        Hacer Clic en los botones de servicio correspondientes

      4. Iniciar los Apache y MySql como servicios de windows:
        Hacer Clic en los botones Start del servicio correspondientes

  • 2. Configurar PHPMyadmin arrow_drop_down

      1. Abrir la cónsola de php myadmin desde el epxplorador:

        http://localhost/phpmyadmin

      2. Iniciar sesión con el nombre y usuario
        user: root
        password (en blanco)

      3. Crear una nueva base de datos
        Hacer Clic en la opción "NEW"

      4. Crear base de datos con el nombre supermercadosmax:

  • 3. Utilizar asistente de Exportación/Importación de Duplicator.php arrow_drop_down

    1. Descomprimir los archivos supermercadosmax_cf2a4b83a85c0cff7807180416123917_archive e installer.php dentro de la carpeta del proyecto:
      C:\xampp\htdocs\supermercadosmax

    2. En el explorador navegar hasta la carpeta del proyecto y ejecutar el archivo installer.php


      http://localhost/supermercadosmax/installer.php

      Y hacer click en Next Y eperar que finalice el proceso

    3. En el paso 2, se debe agregar la información de configuración del servidor

      1. Action: Connect and Remove all data
      2. HOST: localhost
      3. Database: supermercadosmax
      4. user: root
      5. password:


    4. Presionar el botón test database y esperar la respuesta del servidor. Si la respuesta es correcta presionar NEXT


      Aceptar el mensaje de confirmación



      Si se genera un error verificar los pasos anteriores

    5. En el paso 3 abrir la pestañas:

      1. New Settings

        en la opción URLCambiar el protocolo de conexión http:// por https://

      2. Opciones / New Admin Account

        cambiar el nombre del usuario administrador por el de tu preferencia. En este ejemplo se utilizará

        Usuariomaster

        Contraseña123456

      3. Opciones / Scan Options

        en la opción Site URLCambiar el protocolo de conexión http:// por https://

      Verificar que el URL y SiteURL comiencen con protocolo https:// para evitar inconvenientes con el inicio de sesión con las redes sociales

      Y hacer click en Next Y eperar que finalice el proceso

    6. En el paso 4 verificar que no hayan errores de instalación


      Y hacer click en Site login y automáticamente será redireccionado a la página de inicio de sesión del sitio web

    7. En la página de inicio de sesión utilizar el nombre de usuario y contraseña especificados en el paso 3.1


    8. Una vez iniciada la sesión serás redirigido al dashboard y deberás eliminar los archivos de instalación


    9. La tienda virtual está lista para ser utilizada.

      Ir a la tienda

  • 4. Errores comunes arrow_drop_down
    • 1. No se muestran las ofertas diarias en el slider principal arrow_drop_down

      Causa:

      No existe una categoría de productos con slug definido como ofertas-diarias o no hay productos asignados a esa categoría.

    • 2. No se muestran las ofertas semanales en la página principal arrow_drop_down
      Causa:

      No se ha utilizado el shortcode:

      [products per_page="4" columns="4" category="Ofertas Semanales" orderby="rand" order="rand" on_sale="true"]

      O no se ha definido una categoría de productos en la opción Categoría de productos en la Sección de productos del personalizador

    • 3. No se muestran las ofertas del mes en la página principal arrow_drop_down

      Causa:

      1. No se ha seleccionado la categoría ofertas semanales

      2. No se hay productos asignados a la categoría ofertas semanales

      3. No se ha seleccionaod la categoría de Ofertas Diárias en la sección en la Sección Slider de productos del personalizador

    • 4. Error en dashboard del plugin NexttendSocialLoginerrorURI arrow_drop_down

      Causa:

      La página utiliza protocolo http en vez de protocolo https.

    • 5. Error de inicio de sesión con cuenta Facebook arrow_drop_down

      Causa:

      La página utiliza protocolo http en vez de protocolo https.

      Solución

      Se debe cambiar la URL del sitio web en ajustes para forzar Ingresar al sitio web con protocolo https:

    • 6. Error de inicio de sesión con cuenta Google+ arrow_drop_down

      Causa:

      La página utiliza protocolo http en vez de protocolo https.

      Solución

      Se debe cambiar la URL del sitio web en ajustes para forzar Ingresar al sitio web con protocolo https:

    • 7. Error al enviar emails desde el formulario de contacto arrow_drop_down

      Causa:

      Por cuestiones de seguridad, no se ha configurado un servicio de envío de emails (SMTP) para esta evaluación.


    • 8. Finalizar compras arrow_drop_down

      Una vez presionado el botón finalizar compras no refrescar el navegador hasta que el proceso termine. Este proceso puede durar unos minutos.


Forzar web a https

Ir a ajustes generale y cambiar el protocolo de conexión por defecto http por https en la opción Dirección del sitio (URL). https://localhost/supermercadosmax/wp-admin/options-general.php

Es necesario forzar el protocolo de conexión a https para evitar inconvenientes y errores a la hora de realizar el inicio de sesión con las redes sociales facebook y google+
